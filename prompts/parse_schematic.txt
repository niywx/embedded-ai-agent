## 输入

## 解析重点

### 1. 引脚识别
- 准确识别 MCU 芯片型号和引脚编号
- 区分物理引脚号和功能引脚名
- 识别复用功能（如 PA0/ADC0）

### 2. 信号跟踪
- 跟踪每个引脚连接到的外部器件
- 识别常见外设：LED、按键、蜂鸣器、传感器、通信接口
- 注意信号命名（NET 名称）

### 3. 电气特性
- **低电平有效 (Active Low)**：识别反相圆圈、"N"后缀、"/LED"表示法
- **高电平有效 (Active High)**：直接连接
- **上拉/下拉电阻**：识别 R 和箭头方向
- **开漏/推挽**：根据电路拓扑判断

### 4. 方向判断
- **输出 (Output)**：驱动 LED、蜂鸣器、继电器等
- **输入 (Input)**：读取按键、传感器信号
- **双向 (Bidirectional)**：I2C、1-Wire 等

### 5. 常见模式识别

#### LED 电路
```
MCU_PIN ----> LED ----> GND  （高电平点亮，output, active_high）
MCU_PIN <---- LED <---- VCC  （低电平点亮，output, active_low）
```

#### 按键电路
```
MCU_PIN ---- KEY ---- GND    （按下为低，input, active_low, pull_up）
MCU_PIN ---- KEY ---- VCC    （按下为高，input, active_high, pull_down）
```

#### 蜂鸣器电路
```
MCU_PIN ----> BUZZER ----> GND  （高电平响，output, active_high）
```

## 特殊标注识别

- `/LED1` 或 `LED1_N` → 低电平有效
- `LED1+` 或 `LED1` → 高电平有效
- `10K↑` → 10K 上拉电阻
- `OD` → 开漏输出
- `PP` → 推挽输出

## 注意事项

1. 如果原理图不清晰，在 special_requirements 中说明
2. 如果无法确定电气特性，标记为 "unknown"
3. 不要猜测未明确显示的连接
4. 保留原理图中的信号命名
5. 对于复杂电路（如驱动电路），在 special_requirements 中详细说明
